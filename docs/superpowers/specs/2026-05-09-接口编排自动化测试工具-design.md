# 接口编排自动化测试工具 — 设计规格

## 1. 概述

建设一个面向 RSH Cloud 流程回归验证的桌面工具。用户通过 6 步向导完成"选账号、选流程、填表单、配节点、运行、看报告"，系统内部负责环境读取、会话管理、流程解析、节点指派、分支并发执行和报告记录。

定位：专业 QA 桌面工具。目标体验：VS Code + Linear 的交叉感——结构清晰、信息密度适中、视觉噪声低、每个操作都有即时反馈。

### 1.1 技术栈

- 桌面框架：PySide6
- 事件循环：qasync（Qt + asyncio 融合）
- HTTP 客户端：httpx.AsyncClient
- 本地存储：SQLite
- 报告输出：HTML（内嵌 CSS 单文件）、JSON
- 图标：Lucide 线性风格 SVG，stroke-width 1.5，20px

### 1.2 项目当前状态

从零开始，尚无实现代码。

## 5. 各步骤详细设计

### 5.1 步骤 1：账号选择

顶部当前会话状态卡片（绿色在线标识 + 用户名 + [查看会话] [切换账号]）。

切换账号面板：实时搜索框 + 部门下拉 + 岗位下拉 + [重置]。下方用户卡片列表（头像占位 + 姓名 + 部门·岗位 + 当前会话标识）。选择账号后立即登录，显示 loading 动画，2 秒内感知反馈。

### 5.2 步骤 2：流程选择

顶部搜索框 + 分类筛选下拉 + [刷新列表]。流程卡片（图标 + 名称 + 分类标签 + 节点数），悬停展开小预览（含并行分支说明）。底部已选择流程确认区。

### 5.3 步骤 3：表单填写

操作栏：[从历史采样] [清空] [保存为模板]。

从历史采样流程：点击按钮 → 弹出侧滑面板 → 按流程类型查询最近 10 个历史实例列表 → 用户选择一个 → 数据自动填充表单字段 → 关闭面板。

表单字段按后端返回顺序展示，必填字段红色星号标记。只读字段灰色不可编辑。

模式切换：自动采样模式 / 手动填写模式。

### 5.4 步骤 4：节点配置

操作栏：[一键指定] [一键随机] [重置全部]。

表格行级编辑：

| # | 节点名称 | 类型 | 指派方式 | 结果 |
|---|---------|------|---------|------|
| 1 | 部门负责人 | 审批 | [手动指定▼] | 欧阳改 |
| 2 | 分管领导 | 审批 | [岗位随机▼] | ⚠ 待配置 |
| 3 | 财务复核 | 审批 | [范围随机▼] | ⚠ 待配置 |

指派方式下拉选项：手动指定 / 一键指定 / 一键随机 / 范围随机 / 岗位随机 / 部门随机。

选择指派方式后，同行右侧展开配置面板：
- 手动指定 → 用户搜索选择器
- 岗位随机 → 岗位多选下拉 + 部门限定 + 排除选项
- 范围随机 → 用户/部门/岗位多选 + 排除规则

未配置必填节点时"下一步"按钮置灰 + 提示"还有 N 个节点未配置"。

### 5.5 步骤 5：运行监控

顶部状态栏：运行中进度条 + 已用时间 + 节点计数 + [暂停] [停止]。

左：分支树（竖向时间线）。节点状态图标：`○` 待处理 `●` 运行中 `✓` 完成 `✗` 失败 `⏸` 暂停。

右：节点详情面板（实时刷新）。显示：URL 方法路径（脱敏）、响应状态、会话使用者、耗时。

运行完成时状态栏变绿，自动跳转步骤 6。

### 5.6 步骤 6：报告查看

操作栏：[HTML] [JSON] [另存为运行计划]。

执行摘要卡片：流程名、状态、分支计数、节点计数、用时、发起人。

分支详情：可折叠树，失败节点显示错误原因。

会话日志：时间线展示登录、重登、SID 缓存命中等事件。

敏感字段在报告中显示为 `****`，鼠标悬停显示脱敏提示。

## 7. 核心模块职责

| 模块 | 职责 |
|------|------|
| EnvironmentService | 读取 `.env` 和 `config/` 下非敏感配置 |
| SecretProvider | 只读本地 `.env` 敏感字段，内存持有，不写日志不落库 |
| SessionManager | 全局单例，缓存键 `env_id+login_type+username`，内部 per-key `asyncio.Lock`，对外 `get_session()`/`refresh_if_needed()`/`mark_invalid()` |
| FlowCatalogService | 用发起人 SID 查询可发起流程，返回 `StartableFlow` 列表 |
| FlowInspector | 解析流程定义，返回 `FlowNodeDescriptor` 列表 + 分支树 + 表单字段 |
| AssignmentEngine | 纯计算，输入候选池+策略，输出 `AssignmentResult`（含种子、快照ID、原因） |
| BranchScheduler | 管理分支并发，`asyncio.Semaphore` 控制上限，默认 `wait_all_then_fail` |
| NodeExecutor | 单节点执行器：获取处理人→获取会话→发请求→解析结果→写记录 |
| RunRecorder | 统一写入 SQLite，脱敏在写入层统一处理 |
| ReportExporter | 读取 SQLite 生成 HTML / JSON 报告 |

## 8. 数据模型

### 8.1 RunPlan（内部契约）

```yaml
run_plan_version: 1
name: "付款流程回归-默认场景"
environment:
  env_id: "test"
  gateway_ref: "config.env.GATEWAY_URL"
starter:
  mode: "default_super_account"
  user:
    username: "欧阳改"
    display_name: "欧阳改"
flow:
  flow_id: "flow_xxx"
  flow_name: "付款审批流程"
form_data:
  strategy: "fixed"
  fields:
    amount: 12000
    reason: "自动化回归测试"
assignments:
  default_policy: "manual_first"
  random_seed: 20260509
  rules:
    - target:
        node_id: "dept_approve"
        field: "approver"
      selector_type: "user"
      mode: "position_random"
      scope:
        departments: ["财务部"]
        positions: ["部门负责人"]
        exclude_starter: true
        exclude_used: true
branch_execution:
  mode: "parallel_branches"
  max_branch_concurrency: 5
  failure_policy: "wait_all_then_fail"
session_policy:
  sid_reuse: true
  relogin: "only_when_missing_or_expired"
  duplicate_login_guard: true
```

### 8.2 SessionLease

```python
@dataclass
class SessionLease:
    env_id: str

    username: str
    sid: str
```

### 8.3 SQLite 表

- `run_plan`：id, name, version, env_id, starter_username, flow_id, flow_name, random_seed, plan_json, created_at
- `execution_run`：id, run_plan_id, flow_instance_id, status, started_at, finished_at, failure_policy
- `branch_run`：id, execution_run_id, branch_key, status, started_at, finished_at
- `node_run`：id, branch_run_id, node_id, node_name, status, assignee_snapshot, started_at, finished_at, retry_count
- `request_log`：id, node_run_id, method, url, request_summary, response_summary, status_code, duration_ms, created_at
- `session_log`：id, execution_run_id, env_id, login_type, username, generation, action, reason, created_at
- `candidate_snapshot`：id, execution_run_id, node_id, field, selector_type, candidates_json, created_at

## 9. 脱敏策略

以下字段在写入数据库和报告前必须脱敏：`password`, `token`, `sid`, `cookie`, `authorization`, `client_secret`, 数据库密码, Nacos token。

脱敏在 `RunRecorder.write()` 层统一执行，禁止各模块自行处理。

## 10. 表单历史采样

用户点击"从历史采样" → 系统用当前账号 SID 按**流程类型**查询该类型最近 10 个已完成实例 → 弹出侧滑列表 → 用户选择一个 → 系统拉取该实例的表单数据 → 填充当前表单字段 → 关闭面板。

## 11. MVP 技术完成标准

1. EnvironmentService 能读取开发环境配置。
2. SecretProvider 能读取本地密钥，不把密钥写入仓库。
3. SessionManager 能按处理人自动登录、缓存 SID、复用同一账号 SID，并在 SID 失效时重登。
4. 默认"欧阳改"能查询可发起流程。
5. 指定账号后能查询该账号可发起流程。
6. FlowInspector 能解析流程节点和并行分支。
7. AssignmentEngine 能完成手动、一键、随机、岗位、部门指派。
8. BranchScheduler 能按分支并发执行。
9. RunRecorder 能保存运行计划、分支、节点和会话日志。
10. ReportExporter 能生成基础 HTML 或 JSON 报告。
