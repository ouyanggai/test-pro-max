# UI 全面重构规格文档

> 日期：2026-05-10
> 目标：重构 PySide6 桌面应用的 UI，使其现代化、可操作、错误可见

---

## 1. 概述

现有 UI 问题：
- 按钮/输入框样式粗糙，无现代感
- 搜索功能全是 `pass`，不可用
- 步骤间「下一步」在底部，视觉引导弱
- 无错误提示，用户不知道发生了什么
- 无日志系统，排查问题困难

重构目标：
- 现代化 UI（Qt Fusion 主题 + 自定义 CSS）
- 功能完整可用（账号搜索/流程加载/节点配置）
- 步骤验证 + 智能导航
- 错误实时可见 + 结构化日志

## 3. 各步骤设计

### Step 1：账号选择

**布局：**
```
┌─────────────────────────────────────┐
│  发起账号                            │
│                                      │
│  [ 欧阳改                    🔍 ]   │  ← 搜索输入框，下拉候选
│                                      │
│  ✅ 已选择：欧阳改                    │  ← 绿色确认行
│                                      │
│  [     ← 上一步     ]  [ 下一步 → ] │  ← 居中偏下
└─────────────────────────────────────┘
```

**行为：**
- 输入框默认预填「欧阳改」（从 `.env` 的 `DEFAULT_USERNAME`）
- 实时从 `/web/user/api/user/list` 模糊匹配，过滤结果显示在下拉列表
- 下拉列表最多显示 10 条，键盘 ↑↓ 选择，Enter 确认
- 用户修改输入后，`shared_data["starter_username"]` 更新
- 无需「切换账号」按钮——输入即切换

**验证：**
- 输入框非空即通过，模糊匹配即有效

### Step 2：流程选择

**布局：**
```
┌─────────────────────────────────────┐
│  选择流程                            │
│  [ 搜索流程名称...            🔍 ]   │
│                                      │
│  ┌──────────────┐ ┌──────────────┐ │
│  │ ● 费用报销   │ │ ○ 采购申请   │ │
│  │   行政 > 通用│ │   采购 > 办公│ │
│  └──────────────┘ └──────────────┘ │
│                                      │
│  未选择流程                         │
│  [     ← 上一步     ]  [ 下一步 → ] │
└─────────────────────────────────────┘
```

**行为：**
- 页面加载时自动调用 `/web/flowTemplateApi/list`
- 搜索框 `keyword` 参数实时过滤
- 卡片点击选中（单选），蓝色边框高亮
- 选中后 `shared_data["flow_id"]` + `shared_data["flow_name"]` 更新

**验证：**
- 选中一个流程才可进入下一步
- 未选时点击「下一步」：自动滚动到流程列表顶部，卡片网格抖动 + 高亮，提示「请先选择一个流程」

### Step 3：表单填写

**布局：**
```
┌─────────────────────────────────────┐
│  填写表单                            │
│  [从历史采样] [清空] [保存模板]     │

**验证：**
- 所有 `required=true` 字段必填
- 未填时点击「下一步」：第一个缺失字段高亮 + 抖动动画，红色提示文字「请填写 [字段名]」

### Step 4：节点配置

**布局：**
```

- 点击「下一步」时检查所有行，若有「⚠待配置」：自动滚动到该行，行高亮，提示文字显示

## 4. 错误展示系统

### 4.1 界面内错误通知

**错误通知条**（位于内容区顶部，内容区外）：
```
┌──────────────────────────────────────────────────────────┐
│ ⚠ 账号登录失败：用户名或密码错误       [查看详情] [✕] │
└──────────────────────────────────────────────────────────┘
```
- 红色背景条，出现时自动弹出
- 「查看详情」展开完整错误信息
- 「✕」手动收起
- 可叠加多条错误（垂直堆叠）

**错误场景：**
| 场景 | 显示内容 |
|------|---------|
| 登录失败 | 错误码 + 响应 message |
| API 请求失败 | 请求路径 + HTTP 状态码 + 错误信息 |
| 流程加载失败 | 错误信息 + 「重试」按钮 |
| 节点指派失败 | 节点名 + 失败原因 |

### 4.2 状态指示

- 账号状态：绿色「●」= 已登录 / 灰色「○」= 未登录
- 流程加载：右上角显示加载动画 spinner
- 运行状态：进度条 + 节点树实时更新

## 5. 日志系统

### 5.1 日志配置

| 配置项 | 值 |
|--------|-----|
| 日志目录 | `~/.workflow_test_desktop/logs/` |
| 文件命名 | `app_YYYY-MM-DD.log` |
| 保留策略 | 保留最近 7 天，超期自动删除 |
| 日志级别 | DEBUG / INFO / WARNING / ERROR |
| 格式 | `[时间] [级别] [模块] 消息` |

### 5.2 日志内容

- **INFO**: 启动完成、登录成功、流程加载成功、运行开始/完成
- **WARNING**: 重试操作、响应超时（非致命）
- **ERROR**: 登录失败、API 错误、运行异常（含完整堆栈）

### 5.3 日志查看器

主窗口右上角 **📋 日志** 按钮，点击弹出 `LogViewerDialog`：
- 实时显示最新日志（自动滚动）
- 级别过滤器：全部 / ERROR / WARNING / INFO
- 搜索框过滤日志内容
- 「打开日志目录」按钮

## 9. API 接入清单

| Step 5 | `POST /flowJobTaskLink/findPendingByFlowInstanceId` | 查询待办任务 |
| Step 5 | `POST /web/flowInstanceApi/audit` | 执行审批 |
