# Claude Code 技能速查

## 新项目启动顺序

```
/setup-matt-pocock-skills  →  /grill-with-docs  →  /to-prd  →  /to-issues
```

## 技能速查表

| 技能 | 用途 | 触发词 |
|------|------|--------|
| `/setup-matt-pocock-skills` | 初始化配置（本地 Markdown 任务推进、领域文档） | 首次使用前必跑 |
| `/grill-with-docs` | 定义领域术语，生成 CONTEXT.md | "压力测试计划" |
| `/grill-me` | 无情面试你的设计 | "grill me" |
| `/to-prd` | 从上下文生成 PRD | "生成 PRD" |
| `/to-issues` | PRD 拆成本地 Markdown 任务 | "拆分任务" |
| `/tdd` | 测试驱动开发 | "TDD"、"红绿重构" |
| `/prototype` | 快速验证设计 | "原型"、"试试几种" |
| `/diagnose` | 调试复杂 Bug | "diagnose this"、"调试" |
| `/improve-codebase-architecture` | 重构，提升可测试性 | "改善架构" |
| `/triage` | 本地任务状态流转 | "分类任务" |
| `/zoom-out` | 看代码大局 | "大局视角" |
| `/write-a-skill` | 创建新技能 | "写技能" |

## 日常工作流

```
/grill-me    →  /to-prd  →  /to-issues  →  /tdd（逐个实现）
                  设计验证      规划          拆分         编码
```

## 已安装的 Matt Pocock 技能

| 技能 | 说明 |
|------|------|
| `setup-matt-pocock-skills` | 基础配置（本地 Markdown 任务推进、领域文档） |
| `grill-me` | 代码审查/提问，无情面试你的设计 |
| `grill-with-docs` | 带文档的代码审查，定义领域术语生成 CONTEXT.md |
| `improve-codebase-architecture` | 改进代码架构，提升可测试性 |
| `tdd` | 测试驱动开发 |
| `to-prd` | 从上下文生成产品需求文档 |
| `to-issues` | 将 PRD 拆分成本地 Markdown 任务 |

## 关键文件

- `CONTEXT.md` - 领域术语定义（/grill-with-docs 生成）
- `docs/adr/` - 架构决策记录
- `docs/开发文档/issues/` - 本地 Markdown 任务推进目录
